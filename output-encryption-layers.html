Sulphur-2 — Output Handling & Encryption Layers

Implemented architecture: two optional encryption layers between browser, Flask, and RunPod.

Two independent toggles:
  • Browser-side per-field encryption — triggered when the frontend sends "encryption": true. The browser fetches /api/crypto-key, encrypts image/prompt/keyframes with AES-256-GCM, and Flask decrypts them before building the RunPod payload.
  • Server-side full-payload encryption — triggered whenever COMFY_ENCRYPTION_KEY is configured. Flask serializes the entire input dict and encrypts it as input.encrypted. The RunPod worker handler decrypts it before ComfyUI runs.
Output images are encrypted by the worker whenever either layer is active.

1. Multiple OUTPUT_NODE = True nodes

RunPod's handler reads the full ComfyUI /history/{prompt_id} record. Every node with OUTPUT_NODE = True that produced a ui.images entry has its files collected. They all end up in a single array:

{ "status": "COMPLETED", "output": { "images": [ { "filename": "sulphur_output_00001.mp4", "type": "s3_url", "data": "https://.../video1.mp4?..." }, { "filename": "sulphur_output_00002.mp4", "type": "s3_url", "data": "https://.../video2.mp4?..." }, { "filename": "enhanced_prompt.txt", "type": "base64", "data": "YSBjYXQg..." } ] } }
In Sulphur-2 workflows, only one output node is intentionally wired: the RunpodVideoBridge node after VHS_VideoCombine. Multiple output nodes work, but the backend's download_job_files() skips .txt files, so extra outputs would need to be consumed elsewhere.

2. Must the output dict use ui.images?

Yes, for the standard RunPod handler. RunPod's handler is specifically coded to look at the ComfyUI convention ui.images (a list of {filename, subfolder, type} dicts). It then resolves each file from /comfyui/output/ (or temp/input), uploads to S3/R2 if configured, and returns the array shown above.

Key Recognized by RunPod handler? Notes
ui.images Yes Standard ComfyUI image/video output; handler uploads & returns it.
ui.urls No (standard image) Custom key; standard handler ignores it unless you patch it.
ui.data No Not a standard ComfyUI output key.
result No Used for ComfyUI node graph data flow, not surfaced to the client.
# What RunpodVideoBridge returns to ComfyUI { "ui": { "images": [ {"filename": "sulphur_output_00001.mp4", "subfolder": "", "type": "output"} ] }, "result": (images,) # passed to downstream nodes if any }

3. Encryption layers

All AES-256-GCM encryption uses the COMFY_ENCRYPTION_KEY environment variable (64 hex chars = 32 bytes). The wire format is base64(nonce(12 bytes) + ciphertext + GCM tag(16 bytes)).

What exactly is encrypted?

Thing Encrypted? Where / When
Browser → Flask request body Optionally Only when the frontend sets encryption: true. Image, prompt/keyframe text, and previous frame are encrypted with AES-256-GCM.
RunPod input payload Yes (if COMFY_ENCRYPTION_KEY set) Flask serializes the entire input dict and encrypts it as input.encrypted before POST to RunPod.
Workflow JSON + images inside RunPod No (at runtime) The worker handler decrypts before passing to ComfyUI, so ComfyUI sees plaintext.
Filename (sulphur_output_00001.mp4) No Sent/stored in plaintext in output.images[].filename.
S3/R2 signed URL No The URL is plaintext; it grants time-limited access to the object.
File content returned by RunPod Yes (if encryption active) The worker encrypts each output image/video when input.encryption == true or input.encrypted is present.
File content downloaded from RunPod → Flask Yes, then decrypted download_job_files() decrypts bytes when img.encrypted == true.
Stored local MP4 on disk No After decryption, files are written to /app/downloads/<job_id>/ as plain MP4s.
COMFY_ENCRYPTION_KEY itself Yes (must stay secret) Loaded from env var only. Exposed to the browser via /api/crypto-key only when browser-side encryption is enabled.

6. Bottom line

  • Browser-side encryption is optional; when enabled, the browser fetches the key and encrypts image/prompt/keyframes.
  • Server-side full-payload encryption is automatic whenever COMFY_ENCRYPTION_KEY is set.
  • The two layers can be active independently or together; either one causes the worker to encrypt output files.
  • RunPod's handler collects ui.images from all OUTPUT_NODE nodes and returns them.
  • Filenames and signed URLs remain plaintext; encryption protects input payload bytes and output file bytes.
