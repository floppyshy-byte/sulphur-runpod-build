Sulphur-2 — Target Output Handling & Encryption Layers

Target architecture: encryption happens only between Flask and RunPod; the browser sends plaintext.

This page describes the target architecture, not the current implementation. Browser-side encryption and the /api/crypto-key endpoint will be removed. The COMFY_ENCRYPTION_KEY secret will live only in server-side environment variables.

1. Multiple OUTPUT_NODE = True nodes

RunPod's handler reads the full ComfyUI /history/{prompt_id} record. Every node with OUTPUT_NODE = True that produced a ui.images entry has its files collected. They all end up in a single array:

{ "status": "COMPLETED", "output": { "images": [ { "filename": "sulphur_output_00001.mp4", "type": "s3_url", "data": "https://.../video1.mp4?..." }, { "filename": "sulphur_output_00002.mp4", "type": "s3_url", "data": "https://.../video2.mp4?..." }, { "filename": "enhanced_prompt.txt", "type": "base64", "data": "YSBjYXQg..." } ] } }
In Sulphur-2 workflows, only one output node is intentionally wired: the RunpodVideoBridge node after VHS_VideoCombine. Multiple output nodes work, but the backend's download_job_files() skips .txt files, so extra outputs would need to be consumed elsewhere.

2. Must the output dict use ui.images?

Yes, for the standard RunPod handler. RunPod's handler is specifically coded to look at the ComfyUI convention ui.images (a list of {filename, subfolder, type} dicts). It then resolves each file from /comfyui/output/ (or temp/input), uploads to S3/R2 if configured, and returns the array shown above.

Key Recognized by RunPod handler? Notes
ui.images Yes Standard ComfyUI image/video output; handler uploads & returns it.
ui.urls No (standard image) Custom key; standard handler ignores it unless you patch it.
ui.data No Not a standard ComfyUI output key.
result No Used for ComfyUI node graph data flow, not surfaced to the client.
# What RunpodVideoBridge returns to ComfyUI { "ui": { "images": [ {"filename": "sulphur_output_00001.mp4", "subfolder": "", "type": "output"} ] }, "result": (images,) # passed to downstream nodes if any }

3. Target encryption layers

Encryption is optional and controlled by the COMFY_ENCRYPTION_KEY environment variable (64 hex chars = 32 bytes). The wire format is base64(nonce(12 bytes) + ciphertext + GCM tag(16 bytes)).

In the target architecture, the browser and Flask live on the same host, so the browser sends plaintext. The key is never exposed over HTTP. Encryption is applied by Flask to the entire RunPod input dict, and decrypted by the RunPod worker handler before ComfyUI runs.

What exactly is encrypted?

Thing Encrypted? Where
Browser → Flask request body No Browser and Flask are on the same host. HTTPS protects transport from eavesdroppers.
RunPod input payload Yes (if COMFY_ENCRYPTION_KEY set) Flask serializes the entire input dict and encrypts it before POST to RunPod.
Workflow JSON + images inside RunPod No (at runtime) The worker handler decrypts before passing to ComfyUI, so ComfyUI sees plaintext.
Filename (sulphur_output_00001.mp4) No Sent/stored in plaintext in output.images[].filename.
S3/R2 signed URL No The URL is plaintext; it grants time-limited access to the object.
File content at rest in S3/R2 Maybe Only if the RunPod worker is later patched to encrypt uploads. Standard worker does not.
File content downloaded from RunPod → Flask Maybe download_job_files() decrypts if img.encrypted == true (kept for future use).
Stored local MP4 on disk No After decryption, files are written to /app/downloads/<job_id>/ as plain MP4s.
COMFY_ENCRYPTION_KEY itself Yes (must stay secret) Loaded from env var only. Never sent to browser or logged.

6. Bottom line

  • Browser sends plaintext to Flask (same host).
  • COMFY_ENCRYPTION_KEY is server-side only; no /api/crypto-key route.
  • Flask encrypts the entire RunPod input dict when the key is set.
  • RunPod worker handler decrypts before ComfyUI execution.
  • RunPod's handler still collects ui.images from all OUTPUT_NODE nodes.
  • Filenames and signed URLs remain plaintext; encryption protects the input payload bytes.
